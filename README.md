# GigaAM Transcriber

Локальное веб-приложение для транскрибации аудио на русском языке. Работает полностью офлайн после первоначальной загрузки модели. Никакие данные не отправляются на внешние серверы.

**Используемая модель:** [GigaAM v2](https://github.com/salute-developers/GigaAM) от SberDevices — одна из лучших open-source моделей для распознавания русской речи.

## Возможности

- Транскрибация аудио на **русском языке** с высокой точностью
- Поддержка аудиоформатов: **MP3, WAV, M4A, OGG, FLAC, OPUS**
- Поддержка видеоформатов: **MP4, MKV, AVI, MOV, WEBM, TS** — аудиодорожка извлекается автоматически
- До **10 файлов** за раз, до **500 МБ** каждый
- Поддержка **видеофайлов**: MP4, MKV, AVI, MOV, WEBM — аудиодорожка извлекается автоматически
- Три формата вывода: **TXT, SRT** (субтитры с тайм-кодами), **JSON**
- **VAD-сплит** — умная нарезка по паузам в речи (Silero VAD), не режет слова
- Инлайн-просмотр результатов прямо в браузере, копирование и скачивание
- Полностью **локальный инференс** — данные не покидают компьютер
- Тёмный минималистичный интерфейс

## Установка

### Windows

#### Шаг 1 — Установить Python 3.11

Скачайте установщик с [python.org](https://www.python.org/downloads/release/python-3119/) или через winget:

```powershell
winget install Python.Python.3.11
```

Убедитесь, что Python 3.11 доступен:

```powershell
py -3.11 --version
# Python 3.11.x
```

#### Шаг 2 — Установить ffmpeg

ffmpeg нужен для чтения MP3, M4A и других форматов:

```powershell
winget install Gyan.FFmpeg
```

После установки **закройте и откройте PowerShell заново**, затем проверьте:

```powershell
ffmpeg -version
# ffmpeg version 7.x ...
```

Если winget недоступен — скачайте вручную:
1. Зайдите на https://www.gyan.dev/ffmpeg/builds/
2. Скачайте `ffmpeg-release-essentials.zip`
3. Распакуйте, например, в `C:\ffmpeg`
4. Добавьте путь в PATH:

```powershell
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\ffmpeg\bin", "User")
```

Перезапустите PowerShell и проверьте `ffmpeg -version`.

#### Шаг 3 — Клонировать репозиторий

```powershell
git clone https://github.com/ВАШ_ЛОГИН/gigaam-transcriber.git
cd gigaam-transcriber
```

Или скачайте ZIP с GitHub и распакуйте.

#### Шаг 4 — Создать виртуальное окружение

```powershell
py -3.11 -m venv .venv
.venv\Scripts\activate
```

Вы увидите `(.venv)` перед строкой ввода — это значит, окружение активно.

#### Шаг 5 — Установить зависимости

```powershell
pip install fastapi "uvicorn[standard]" python-multipart pydub silero-vad ffmpeg-python python-dotenv aiofiles numpy
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cpu
pip install git+https://github.com/salute-developers/GigaAM.git
```

> Если у вас есть NVIDIA GPU — установите torch с поддержкой CUDA вместо CPU-версии:
> ```powershell
> pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu121
> ```

#### Шаг 6 — Создать файл конфигурации

```powershell
copy .env.example .env
```

## Запуск

Активируйте виртуальное окружение (если ещё не активно):

```bash
# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

Запустите сервер:

```bash
uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

Откройте браузер и перейдите по адресу:

```
http://localhost:8000
```

> Флаг `--reload` автоматически перезапускает сервер при изменении файлов.
> Для продакшена уберите `--reload` и укажите `--workers 1`.

## Использование

### Этап 1 — Установка модели

При первом открытии приложение покажет экран установки модели.

1. Нажмите кнопку **«Скачать и установить модель»**
2. Дождитесь загрузки весов (~900 МБ). Прогресс отображается в реальном времени
3. После завершения приложение автоматически перейдёт к следующему этапу

Веса сохраняются в `~/.cache/gigaam` и загружаются только один раз. При следующем запуске модель загрузится из кеша мгновенно.

> Во время загрузки нужен интернет. После — приложение работает полностью офлайн.

---

### Этап 2 — Загрузка файлов

**Способ 1 — Drag & Drop:**
Перетащите один или несколько аудиофайлов прямо в обозначенную зону на странице.

**Способ 2 — Через диалог:**
Нажмите на зону загрузки, выберите файлы в открывшемся окне.

Поддерживаемые форматы: `mp3`, `wav`, `m4a`, `ogg`, `flac`, `opus`

Ограничения:
- Не более **10 файлов** за раз
- Не более **500 МБ** на файл

После выбора файлов справа появится панель настроек.

---

### Этап 2 — Настройки транскрибации

| Параметр | Описание |
|----------|----------|
| **Формат вывода** | `.txt` — сплошной текст; `.srt` — субтитры с тайм-кодами; `.json` — структурированные данные |
| **Размер чанка** | На сколько секунд нарезать аудио перед подачей в модель. Меньше = точнее на паузах, больше = быстрее |
| **VAD-сплит** | Умная нарезка по паузам в речи. Не режет слова посередине. Рекомендуется оставить включённым |
| **Устройство** | CPU или CUDA (GPU). CUDA видна только если PyTorch определил совместимую видеокарту |

После настройки нажмите **«Начать транскрибацию»**.

---

### Этап 2 — Прогресс обработки

Во время обработки панель настроек заменяется блоком прогресса:

- Текущий обрабатываемый файл
- Прогресс по чанкам (сколько фрагментов обработано)
- Общий прогресс по всем файлам

Кнопка **«Отменить»** немедленно прекращает обработку.

---

### Этап 3 — Результаты

После завершения отображаются карточки с результатами — по одной на каждый файл.

**Каждая карточка содержит:**
- Имя файла, длительность, количество слов, формат
- Кнопку **«Копировать»** — копирует весь текст в буфер обмена
- Кнопку **«Скачать»** — скачивает файл в выбранном формате
- Кнопку **«Развернуть»** — показывает полный текст прямо на странице со скроллом

**В развёрнутом виде:**
- Для `.txt` — сплошной текст с возможностью выделения
- Для `.srt` — блоки с тайм-кодами (формат `ЧЧ:ММ:СС,МСК --> ЧЧ:ММ:СС,МСК`)
- Для `.json` — подсвеченный синтаксис с ключами, строками и числами
- Кнопка **«Копировать всё»** в нижней части
- Статистика: количество слов и символов

Кнопка **«Обработать ещё»** в правом верхнем углу возвращает к загрузке файлов без перезагрузки страницы.

## Форматы вывода

### TXT
Сплошной текст, все фрагменты объединены пробелами.

```
привет это тестовая транскрибация аудиофайла с помощью модели гига ам
```

### SRT
Субтитры с временны́ми метками. Совместим с большинством видеоплееров и редакторов.

```srt
1
00:00:00,000 --> 00:00:05,120
привет это тестовая транскрибация

2
00:00:05,120 --> 00:00:10,340
аудиофайла с помощью модели гига ам
```

### JSON
Структурированные данные — удобно для дальнейшей обработки скриптами.

```json
{
  "filename": "audio.mp3",
  "chunks": [
    { "start": 0.0, "end": 5.12, "text": "привет это тестовая транскрибация" },
    { "start": 5.12, "end": 10.34, "text": "аудиофайла с помощью модели гига ам" }
  ],
  "full_text": "привет это тестовая транскрибация аудиофайла с помощью модели гига ам"
}
```

## Переменные окружения

Файл `.env` в корне проекта (создаётся из `.env.example`):

```env
# Количество параллельных воркеров для инференса
# При нехватке RAM уменьшите до 1
MAX_WORKERS=2

# Вариант модели GigaAM
# Доступные: v2_ctc, v2_rnnt, v1_ctc, v1_rnnt, ctc, rnnt
# v2_ctc — рекомендуется (быстрее, меньше памяти)
# v2_rnnt — точнее, но медленнее
GIGAAM_MODEL=v2_ctc

# Папка для временных файлов и результатов
TMP_DIR=./tmp

# Время хранения результатов в секундах (3600 = 1 час)
TMP_TTL_SECONDS=3600

# Адрес и порт сервера
HOST=0.0.0.0
PORT=8000
```

## Возможные проблемы

### ffmpeg не найден

```
RuntimeError: Couldn't find ffprobe or avprobe
```

**Решение:** установите ffmpeg и убедитесь, что он доступен в PATH:

```bash
ffmpeg -version
```

Если команда не найдена — перезапустите терминал после установки.

---

### Ошибка «Too long wav file»

```
ValueError: Too long wav file, use 'transcribe_longform' method.
```

Модель `v2_ctc` принимает чанки не длиннее 25 секунд. Приложение автоматически ограничивает чанки до 15 секунд, но VAD-сплит может создавать более длинные фрагменты если в записи нет пауз.

**Решение:** отключите VAD-сплит в настройках — фиксированная нарезка всегда соблюдает лимит.

---

### Out of Memory (нехватка RAM)

```
RuntimeError: [WinError 8] Недостаточно памяти
```

**Решение:** уменьшите количество воркеров в `.env`:

```env
MAX_WORKERS=1
```

Каждый воркер загружает свою копию модели (~1.5 ГБ RAM).

---

### Ошибка «Model not found»

```
ValueError: Model 'e2e_ctc' not found. Available model names: [...]
```

Указан несуществующий вариант модели. Откройте `.env` и исправьте:

```env
GIGAAM_MODEL=v2_ctc
```

---

### VAD не работает — falling back to fixed split

Это предупреждение, не ошибка. Silero VAD не смог загрузиться или обработать файл, и приложение автоматически переключилось на фиксированную нарезку. Транскрибация продолжится — просто без «умной» нарезки по паузам.

---

### Сервер запускается, но страница не открывается

Убедитесь что:
1. Виртуальное окружение активировано (видно `(.venv)` в начале строки)
2. Вы запускаете команду из корня проекта (папка `gigaam-transcriber`)
3. Порт 8000 не занят другим процессом:

```powershell
# Windows
netstat -ano | findstr :8000

# macOS / Linux
lsof -i :8000
```

Если порт занят — измените порт в команде запуска:

```bash
uvicorn backend.main:app --host 0.0.0.0 --port 8001 --reload
```

---

### FutureWarning от torch.load

```
FutureWarning: You are using torch.load with weights_only=False
```

Это предупреждение от PyTorch, не ошибка. Модель GigaAM загружается корректно. Предупреждение исчезнет когда SberDevices обновят формат сохранения весов.

## FAQ

**Q: Работает ли приложение без интернета?**
A: Да, после первой загрузки модели (~900 МБ) всё работает полностью офлайн.

**Q: Какой язык поддерживается?**
A: Модель GigaAM обучена на русском языке. Другие языки не поддерживаются.

**Q: Насколько точна транскрибация?**
A: GigaAM v2 показывает Word Error Rate (WER) около 4–7% на чистой речи. Качество снижается при сильном шуме, акценте или нескольких говорящих одновременно.

**Q: Сколько времени занимает транскрибация?**
A: На CPU — примерно 1–2 минуты на каждую минуту аудио. На GPU (CUDA) — в 5–10 раз быстрее.

**Q: Можно ли запустить на GPU?**
A: Да. Установите torch с поддержкой CUDA и выберите «CUDA» в настройках интерфейса (кнопка отображается только если GPU обнаружен). Требования: NVIDIA GPU с минимум 4 ГБ VRAM.

**Q: Результаты сохраняются между сессиями?**
A: Нет. Результаты хранятся в памяти и папке `./tmp` и удаляются через 1 час (настраивается через `TMP_TTL_SECONDS`). Скачайте нужные файлы сразу после обработки.

**Q: Можно ли транскрибировать видео?**
A: Да. Просто перетащите видеофайл (MP4, MKV, AVI, MOV, WEBM, TS) в зону загрузки — аудиодорожка извлекается автоматически через ffmpeg. Видеофайлы отображаются с жёлтой иконкой.

**Q: Почему Python 3.14 не поддерживается?**
A: Пакет `sentencepiece` (зависимость GigaAM) не имеет pre-built wheel для Python 3.14 и не компилируется без cmake + MSVC. Используйте Python 3.11 или 3.12.
